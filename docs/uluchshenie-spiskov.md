# Улучшать списки

Флажок **Параметры → Комфорт → Улучшать списки**. По умолчанию **включён**.

---

## Фильтры по подстроке в списках

### Многословный фильтр

Текст фильтра дробится на фрагменты по пробелам; для совпадения требуется вхождение **каждого** фрагмента (с мягким учётом порядка), совпадения подсвечиваются.

**Где работает:** список информационных баз, быстрая схема модуля, списки автодополнения, панели «Последние места» и «Наборы объектов», панель «Результаты поиска» — и как базовый режим везде, где ниже указан иерархический фильтр.

### Иерархический фильтр

Расширяет многословный для списков с уровнями (тип объекта → сам объект). Точка в тексте фильтра (вне кавычек) делит его на **секции** — по одной на уровень. Секции сравниваются с полным именем элемента **с конца**: последняя секция — с именем самого элемента, предыдущая — с именем его родителя, и т. д.

**Где работает:** панель «Навигатор», диалог выбора объекта, диалог выбора типа (включая поле «Тип» в мастере «Новый реквизит»).

Подробнее — [Общие механизмы → Фильтры по подстроке](obshchie-mekhanizmy.md#filtry-po-podstroke-v-spiskah).

## Автодополнение BSL

- Умный фильтр popup — текст дробится на фрагменты по пробелам, фильтруется по мере ввода
- Панель с флажком **Фильтр** и подписью **Родитель:**
- **Ctrl+Space** при открытом popup — переключение фильтра
- Дополнение от ИР (при подключённом приложении ИР) — строки ИР в списке, HTML-описание

Подробнее — [Автодополнение](avtodopolnenie.md).

---

## Результаты поиска

- Агрегация вхождений всех потомков при выборе группового узла
- Колонка «Путь»
- Счётчик соответствий в подписи узла
- Таблица всех совпадений справа от дерева (поиск по файлам)

Подробнее — [Результаты поиска](rezultaty-poiska.md).

## Поле ввода типа

Улучшенный фильтр в поле **«Тип»** в мастере «Новый реквизит» и диалогах выбора типа. Текст фильтра дробится на фрагменты по пробелам, как и в остальных списках.

---

## Когда применяется изменение

| Механизм | Момент применения |
|----------|-------------------|
| Фильтры в навигаторе, диалогах, автодополнении, поиске | **Сразу** — при следующем вызове фильтра |
| Автодополнение BSL (popup, фильтр, Ctrl+Space) | **Сразу** — при следующем открытии popup |
| Результаты поиска (агрегация, таблица) | **Сразу** — при следующем поиске или обновлении дерева |
| Поле ввода типа | **При следующем старте EDT** |

Поле ввода типа подчиняется флажку только при старте EDT, поэтому изменение флажка требует перезапуска среды для применения к этому механизму.

---

## Отключение

Чтобы отключить все улучшения, снимите флажок **Параметры → Комфорт → Улучшать списки**. При выключении используется штатное поведение EDT.
